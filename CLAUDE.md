# Nathan Portfolio - Project Instructions

## 프로젝트 규칙
- 아래 `policy/` 폴더의 정책 파일들을 반드시 읽고 준수한다.
  - `policy/project.md` - 개발 순서, 브랜치 전략, 배포 환경, DB, 아키텍처, 보안, 성능, 접근성 규칙
  - `policy/coding.md` - 코딩 컨벤션 및 품질 기준
  - `policy/design.md` - 디자인 시스템 및 스타일 규칙
  - `policy/folder.md` - 폴더 구조 및 운영 규칙

## Task 완료 기준
- 코드 구현만으로는 Task 완료가 아니다. 아래 전체를 마쳐야 완료:
  1. 코드 구현
  2. `/code-review` 스킬로 코드 리뷰 → 문제 수정
  3. `/refactoring` 스킬로 리팩터링 검사 → 문제 수정
  4. `todo.md` 체크
  5. 커밋 (ESLint + TypeScript 체크는 hook이 자동 실행)

## 핵심 문서
- `spec/prd.md` - 서비스 기획/유저스토리 기준 문서. 구현 전 의사결정은 이 문서 기준으로 확정
- `spec/tech.md` - 기술스택
- `task/todo.md` - 작업 목록 관리

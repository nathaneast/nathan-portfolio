## CLAUDE.md
- 작업순서 차근히 정의해놓고 순서대로 진행을 부탁하며 티키타카로 진행하기. md문서로 다듬기 고려
- 여기엔 핵심만 넣기 나머지는 스킬이나 훅으로 빼기.
- 개발로 정확한 순서 지시해서 AI 에이전트가 혼자 작업을 시작하지 않도록 CLAIDE.md 파일 작성. 먼저 작업하지말고 나에게 설계안을 주고 확정시 시작

작업순서
1. prd기반 task 만들기
2. 기술 설계 리뷰 (디비, 프론트, 백엔드)
    /spec 폴더에 구현게획 md파일에 남기기
3. AI 에이전트 작업방식 (팀즈, 서브에이전트, 스킬, 훅 등) 리뷰
   클로드코드 문서링크 주입.
    /spec 폴더에 구현게획 md파일에 남기기
4. 디자인, 레이아웃 리뷰
5. 작업시작 - 깃 워킹트리로 각 task에서 작업, 각 task 단위 커밋, todo 체크
  

## 작업방식
- 유저스토리에 따라 구현한 기능에 테스트코드 먼저 작성하고 기능을 구현하는 tdd 방법론으로 개발
- task 단위 개발 끝나고 코드리뷰, 리팩터링 스킬 사용해서 이거까지 완료하는게 task 완료.


## 수정 필요한것
- 클로드 최신문서 기반 내가 지시하면 훅, 스킬, 커맨드, 서브에이전트 만들어주는 스킬
- 실시간으로 토큰양 확인한는것. omc 상태바가 가끔 안보임
- omc 명령어 prefix없이 / 명령어로 바로 사용 가능하게


## 클로드코드 코드관리
- 깃 저장소에 잘 쌓아두고 프로젝트마다 꺼내서 쓸 생각인데 폴더중첩 가능한지?

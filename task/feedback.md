## CLAUDE.md
- 내가 순서대로 말하면 작업순서대로 진행함
- 먼저 진행하지말고 설계 후 나와 논의를 통해 확정하고 개발 시작

- 여기엔 핵심만 넣기 나머지는 스킬이나 훅으로 빼기.

작업순서
1. prd기반 task 만들기
2. 기술 설계 리뷰 (디비, 프론트, 백엔드)
    /spec 폴더에 구현게획 md파일에 남기기
3. AI 에이전트 작업방식 (팀즈, 서브에이전트, 스킬, 훅 등) 리뷰
   클로드코드 문서링크 주입.
    /spec 폴더에 구현게획 md파일에 남기기
4. 디자인, 레이아웃 리뷰
   대략적으로 레이아웃 설계 받기
5. 작업시작 - 깃 워킹트리로 각 task에서 작업, 각 task 단위 커밋, todo 체크


## 주의사항
- 3번 작업 진행시 팀즈로 지시해도 실제 설계는 그렇지 않았음. 클로드코드 문서주고 정확히 지시 or 자동화가 필요
- 개발 배포시 확신하고 복제수준으로 운영 배포할 수 있는 변수 없음
  

## task 단위 작업방식
- 깃 워킹트리로 각 task 버전관리
- 유저스토리에 따라 구현한 기능에 테스트코드 먼저 작성하고 기능을 구현하는 tdd 방법론으로 개발
- task 단위 개발 끝나고 코드리뷰, 리팩터링 스킬 사용해서 이거까지 완료하는게 task 완료. 


## 수정 필요한것
- 클로드 최신문서 기반 내가 지시하면 훅, 스킬, 커맨드, 서브에이전트 만들어주는 스킬
- 깃에 push전에 보안관련 서브 에이전트 실행
- 실시간으로 토큰양 확인한는것. omc 상태바가 가끔 안보임
- 커밋시 convex 스키마, 로직 등 수정사항 발생시 convex도 함꼐 배포

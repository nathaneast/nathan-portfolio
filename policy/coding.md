- 모든 구현은 `TypeScript`를 사용하고 타입을 엄격하게 유지한다.
  - `any` 사용은 금지하고, 불가피한 경우 `unknown` + 타입 가드로 좁힌다.
  - `strict` 모드 기준으로 타입 오류를 남긴 채 병합하지 않는다.
  
- 한 파일은 최대한 `200줄`을 넘기지 않는다.
  - 코드가 길어지면 기능 단위로 파일을 분리한다.
  - 예외는 자동 생성 파일 또는 타입 선언 집중 파일처럼 불가피한 경우에 한정한다. 
  
- 코드가 길어지거나 중복되면 적극적으로 분리/재사용한다.
  - UI는 컴포넌트는 화면의 섹션 단위로 분리한다.
  - 공통 로직은 `utils`(필요 시 `lib`) 폴더 함수로 추출한다.
  - 중복 구현을 허용하지 않고 공통 함수/훅/모듈로 통합한다.
  
- 핵심 비즈니스 로직은 테스트 코드를 반드시 추가한다.
  - 대상: 도메인 규칙, 피드백 캐시 키 정규화, 공개 정책, 환경 분기 가드
  - 테스트 통과 전에는 `dev`/`main` 병합을 진행하지 않는다.
  
- 서버/클라이언트 경계를 명확히 분리한다.
  - API 키, 외부 SDK 호출 등 서버 경계에서만 수행한다.
  
- 에러 처리는 사용자 메시지와 내부 로그를 분리한다.
  - 사용자에게는 이해 가능한 메시지, 내부에는 원인 추적 가능한 구조화 로그를 남긴다.

- 병합 전 기본 품질 체크를 통과해야 한다.
  - `lint`, `typecheck`, 핵심 테스트(비즈니스 로직) 모두 통과해야 한다.
